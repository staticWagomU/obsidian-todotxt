# Product Roadmap 2026 - Obsidian Todo.txt Plugin

## Overview

このドキュメントは、Obsidian Todo.txt Pluginの2026年における製品ロードマップと、1.0.0リリースに向けた機能スコープ、リリース基準を定義します。

## 1.0.0 Release Scope

### Included Features

1.0.0リリースには以下のフェーズで開発された機能が含まれます:

#### Phase 1: MVP (Sprint 1-7)
- **PBI-001**: .txt/.todotxt専用ビュー
  - TextFileViewを継承した専用ビュー実装
  - ファイル拡張子判定による自動ビュー切替
- **PBI-002**: todo.txtパース
  - todo.txt形式の完全パース（優先度、日付、プロジェクト、コンテキスト、タグ）
- **PBI-003**: 完了切替
  - チェックボックスクリックによる完了/未完了切替
  - 完了日の自動設定
- **PBI-004**: 新規タスク作成
  - 「Add Task」ボタンによる新規タスク追加
- **PBI-005**: タスク編集
  - インライン編集機能（テキストフィールド表示）
- **PBI-006**: タスク削除
  - 削除ボタンによるタスク削除機能
- **PBI-007**: ソート表示
  - 優先度・日付による自動ソート

#### Phase 2: フィルタリング&UI拡張 (Sprint 8-12)
- **PBI-008**: 優先度色分けバッジ
  - (A)=赤、(B)=黄、(C)=緑、(D-Z)=青のカラーコーディング
- **PBI-009**: 優先度フィルタ
  - ドロップダウンによる優先度フィルタリング
- **PBI-010**: テキスト検索
  - リアルタイム全文検索機能
- **PBI-011**: グループ化
  - プロジェクト(+)、コンテキスト(@)、優先度によるグループ表示
- **PBI-012**: due:表示
  - due:タグの視覚的表示と期限管理

#### Phase 3: 拡張機能 (Sprint 13-17)
- **PBI-013**: t:グレーアウト
  - t:(threshold)タグによる「いつかやる」タスクの視覚的区別
- **PBI-014**: [[Note]]内部リンク
  - Obsidian内部リンクのクリック可能表示
- **PBI-015**: [text](url)外部リンク
  - Markdown形式の外部リンク対応
- **PBI-016**: rec:繰り返しタスク
  - rec:タグによる繰り返しタスクの自動生成
- **PBI-017**: pri:タグ保存
  - pri:タグと(A-Z)優先度の相互変換

#### Phase 4: UI統合 (Sprint 18-20)
- **PBI-020**: UI統合メガSprint
  - 7機能（完了切替、編集、削除、ソート、フィルタ、グループ化、検索）の統合動作検証
- **PBI-018**: 設定画面
  - デフォルトソート順、フィルタ設定、グループ化設定のUI
- **PBI-019**: 構造化フォーム
  - 新規作成・編集時の構造化入力フォーム

## Release Criteria

1.0.0リリースは以下の4項目をすべて満たした時点で実施します:

### 1. DoD (Definition of Done) 合格
- すべてのPBIがDoD基準を満たしていること
- 自動テストが全てパス (pnpm vitest run)
- リントエラーなし (pnpm lint)
- 型チェックエラーなし (pnpm tsc --noEmit)
- 手動テストでの動作確認完了

### 2. README.md整備
- プラグインの概要説明（日本語・英語両方）
- インストール方法（手動インストール、コミュニティプラグイン）
- 基本的な使い方（スクリーンショット付き）
- todo.txt形式の簡易リファレンス
- ライセンス情報

### 3. manifest.json更新
- バージョン番号を1.0.0に更新
- minAppVersionの適切な設定
- 説明文の最終確認（日本語・英語）
- authorMetadata の整備

### 4. CHANGELOG.md作成
- Phase 1-4の全変更履歴を記載
- v1.0.0リリースノートの作成
- Breaking changesがあれば明記
- 既知の制限事項の記載

## Risk Management

### 技術的リスク
- **Obsidian API変更**: 定期的なAPI変更チェック、minAppVersion適切設定
- **パフォーマンス問題**: 大規模ファイル対応のテスト強化
- **互換性問題**: 複数Obsidianバージョンでのテスト

### プロセスリスク
- **スコープクリープ**: 1.0.0機能凍結、Phase 7以降に延期
- **品質低下**: TDD遵守、DoD厳格適用
- **ドキュメント不足**: Phase 5での集中整備
